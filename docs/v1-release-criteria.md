# DesktopAudioController v1.0.0 출고 기준

## 목적

이 문서는 `v1.0.0`을 단순한 숫자 상승이 아니라, **stable/public 기준점**으로 다루기 위한 출고 조건을 고정합니다.

현재 프로젝트는 `v0.16.1`까지 updater 백업/롤백 보강, 앱별 출력 변경 UX 개선, UWP/Store 앱 아이콘 개선, 설정 백업/복구, 수동 프로필과 사용자 안내 문서 정리를 반영한 상태이며, 이후 `v1.0.0`은 다음 기준을 만족할 때만 발행합니다.

## 현재 전제

```text
- 배포 방식은 portable zip 단일 경로
- 업데이트 방식은 zip 다운로드 -> 압축 해제 -> 기존 실행 폴더 덮어쓰기
- 기본 업데이트 채널은 stable only
- 설정에서 prerelease opt-in 가능
- 저장소 Public 전환 완료
```

## v1.0.0 필수 통과 조건

### 1. 릴리즈 채널 정책 고정

```text
- stable 사용자는 기본값으로 stable 릴리즈만 안내
- prerelease는 opt-in 설정을 켠 경우에만 안내
- 정책이 코드/설정/UI/문서에 일관되게 반영되어 있어야 함
```

### 2. 핵심 사용자 시나리오 회귀 검증 완료

다음 시나리오는 `v1.0.0` 직전 실제 검증 기록이 있어야 합니다.

참조 문서:

```text
docs/regression-checklist.md
```

```text
1. 첫 실행 -> 장치 선택 -> 메인 화면 진입
2. 장치 마스터 볼륨 / 음소거 조작
3. 기본 출력 장치 변경
4. 프로그램 세션 볼륨 / 음소거 조작
5. 프로그램 세션 출력 장치 변경
6. 프로그램 설정 저장 / 다음 실행 시 복원
7. 트레이 최소화 / 복원 / 종료
8. zip 덮어쓰기 업데이트
9. 업데이트 안내 버튼 / 릴리즈 노트 버튼 동작
```

### 3. 배포 산출물 검증 완료

```text
- dotnet build 0 Warning / 0 Error
- 테스트 전체 통과
- publish 스크립트 실행 성공
- zip / sha256 생성 및 검증 성공
- 실환경에서 zip 덮어쓰기 업데이트 재확인
```

### 4. 공개 문서 최소 세트 완료

```text
- README 최신화
- LICENSE 유지
- 실행 화면 스크린샷 유지
- 릴리즈 노트 템플릿 최신화
- 배포/업데이트 방식이 zip 기준으로 명확히 적혀 있어야 함
```

### 5. Public 전환 판단 확정

`v1.0.0`은 stable/public 기준점이므로, 릴리즈 전에는 아래 둘 중 하나가 명시되어야 합니다.

```text
A. 저장소 Public 전환과 함께 v1.0.0 발행
B. Public 전환은 보류하지만, 그 이유와 대체 운영 방식이 명시된 상태에서 stable v1.0.0 발행
```

의사결정 없이 `v1.0.0`만 먼저 발행하는 것은 금지합니다.

## v1.0.0 보류 조건

아래 중 하나라도 남아 있으면 `v1.0.0`을 보류합니다.

```text
- 핵심 시나리오 회귀 검증 미완
- 기본 장치 변경 / 세션 출력 변경 / 트레이 종료 / 설정 복원 경로에 재현 가능한 결함 존재
- zip 덮어쓰기 업데이트 실환경 검증 미완
- 문서가 현재 동작과 불일치
- Public 전환 또는 stable 운영 방식 결정 미완
```

## v1.0.0 비차단 항목

아래 항목은 있으면 좋지만 `v1.0.0`의 필수 조건은 아닙니다.

```text
- GitHub Actions 기반 릴리즈 자동화
- 설치 관리자
- 자동 패처
- advanced 채널 분리 UI
- 입력 장치(마이크) 제어
```

현재 상태는 `v1.0.0` 출고 기준을 정의하는 단계이며, 아직 `출고 승인` 상태는 아닙니다.

```text
- P11-1: 완료
- P11-2: 이 문서로 기준 정의
- P11-3: 후속 작업 필요
- P11-4: 체크리스트 문서화 완료, 실사용 기록 축적 필요
```
