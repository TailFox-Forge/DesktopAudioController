## 게시용 본문

# Windows에서 출력 장치와 앱별 볼륨을 한 번에 관리하는 도구, DesktopAudioController

Windows에서 오디오 장치를 여러 개 쓰다 보면  
"지금 이 장치에서 어떤 프로그램이 소리를 내고 있는지",  
"특정 프로그램만 빠르게 음소거하거나 볼륨을 조절하고 싶은데 너무 번거롭다"  
같은 불편이 자주 생깁니다.

그래서 직접 만든 도구가 **DesktopAudioController**입니다.

이 프로그램은  
**사용자가 고른 출력 장치만 메인 화면에 표시**하고,  
각 장치의 **마스터 볼륨**과  
그 장치에서 실제로 소리를 내는 **프로그램별 세션 볼륨**을  
빠르게 제어할 수 있게 만든 Windows 전용 데스크톱 유틸리티입니다.

---

## 어떤 점이 편한가

### 1. 필요한 출력 장치만 골라서 표시

Windows에는 실제로 쓰지 않는 장치나 가상 장치가 섞여 있는 경우가 많습니다.  
이 프로그램은 설정에서 장치를 선택하면 **선택한 장치만 메인 화면에 표시**합니다.

### 2. 장치 볼륨과 프로그램 볼륨을 한 화면에서 제어

각 장치마다 다음 작업을 바로 할 수 있습니다.

- 마스터 볼륨 조절
- 장치 음소거
- 기본 출력 장치 전환
- 해당 장치에서 출력 중인 프로그램별 볼륨 조절
- 프로그램별 음소거

즉,  
"스피커 전체 볼륨은 유지하고 브라우저만 줄이기",  
"특정 장치에서 재생 중인 앱만 골라서 음소거"  
같은 작업이 훨씬 빨라집니다.

### 3. 조용한 앱도 유지할지, 재생 중인 앱만 볼지 선택 가능

프로그램 세션은 순간적으로 소리가 멈췄다고 목록에서 바로 사라지면 오히려 불편할 수 있습니다.  
그래서 기본값은 **Inactive 세션도 목록에 유지**하고,  
대신 **`현재 재생 중 아님`** 상태로 표시합니다.

필요하면 설정에서 **`재생 중인 앱만 보기`** 옵션을 켜서  
지금 소리가 나는 앱만 따로 볼 수도 있습니다.

### 4. 프로그램 이름을 사용자가 직접 바꿔서 보기 쉽게 정리

같은 실행 파일이 여러 세션으로 보이거나, 표시명이 애매한 경우가 있습니다.  
이럴 때는 **사용자 지정 이름**을 저장해서 더 알아보기 쉽게 정리할 수 있습니다.

### 5. 업데이트와 로그 관리도 현실적으로 정리

최근 버전에서는 아래도 같이 정리했습니다.

- GitHub 릴리즈 기준 업데이트 안내
- 설정창에서 **로그 폴더 바로 열기**
- 공개용 로그에서 **민감 경로/식별자 요약 처리**
- 앱 시작 시 로그 자동 정리
  - **7일 보관**
  - **최대 30개 유지**
  - **총 100MB 상한**

## 설치 / 실행 방법

1. GitHub 릴리즈 페이지에서 최신 zip 파일을 다운로드합니다.
2. 압축을 해제합니다.
3. `DesktopAudioController.exe`를 실행합니다.
4. 첫 실행 시 표시할 출력 장치를 설정합니다.

별도 설치 프로그램 없이 바로 실행하는 방식이라 가볍게 쓰기 좋습니다.
